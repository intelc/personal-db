An imaginary memory system that remembers everything and summarize and reflect and forget is not the solution, rather, we should have a light memory system and a heavy deterministic data stack that leverages the latest in agentic coding.

Every user get their slice of database in the cloud, which syncs to and from with their local client data base. There's 
1. local-sourced data (1 way sync)
2. cloud(or elsewhere)- sourced data (1 way sync)
3. preference data (2 way sync)

agent exists in every client, they are informed
1.what data they have access to (avaliable in the current environment accessible db.)
2.what capability they have access to (cloud can't request for mac to be open a webpage if mac is off)
3. these can be progressively disclosed

very importantly, we have crafts -- which are user created apps that runs inside our environment -> they can be 
1. pure data apps -> user want to extract deterministic insights/ add transformation to some data source (tracker-equivalent)
2. apps with custom code, custom frontend (app-equivalent)
3. pure script apps -> maybe more in the skills land?

our agent will proactively attempt to automate work for user/ draw from existing library

the craft library can be seen as a pip package that allows people to download + dependency management.
